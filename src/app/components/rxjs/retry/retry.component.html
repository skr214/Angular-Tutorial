Retry, RetryWhen, scan and Delay


retry: Resubscribes to the source Observable a specified number of times in case of an error. Useful for handling transient errors.
Example: source$.pipe(retry(3)) → retries the source Observable up to 3 times on error.

retryWhen: Allows customizing the retry logic by providing a notifier Observable. Useful for more complex retry scenarios.
Example: source$.pipe(retryWhen(errors => errors.pipe(delay(1000)))) → retries after 1 second delay on error.

scan: Applies an accumulator function over the source Observable, returning the accumulated result. Useful for maintaining state.
Example: source$.pipe(scan((acc, curr) => acc + curr, 0)) → sums emitted values.

delay: Delays the emissions of the source Observable by a specified timeout. Useful for throttling or debouncing.
Example: source$.pipe(delay(1000)) → delays emissions by 1 second.
